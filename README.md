## Features

- "External" world and avatar ripping
- Automatic renaming and organization based on ID (avtr_, wrld_) and type (.vrca, .vrcw)

## Requirements

- VRChat
- Any VRChat internal that disables encryption
> Just a quick note, this is intended for use with Hexedware's VRChat Internal but this will work with more things than just that
- .Net 9 or greater

## Usage

- Launch SlyRip BEFORE you launch your VRChat.
- That's it! The rest is done for you.

## Release Notes

Before I get people saying that it's broken because it can't find their cache, this assumes you're using the default VRChat cache location. I may change this in the future so you can use a custom cache location.

This is meant to be a simple way for people to get the unencrypted .vrca or .vrcw files without needed to touch the cache themselves. The simplicity is intentional. That's all this is meant for.

I likely won't update this more than I need to but feel free to put in feature requests regardless.
